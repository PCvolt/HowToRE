Git

Git has a plethora of commands, here are the ones I use most.

Getting started and updating a repo

  • git init to initialize git in a directory.
  • git clone <url> to clone a repository in the current directory.
  • git pull --ff-only to update a repository. --ff-only is recommended to avoid any potential merging.

Checking current status

  • git status to show the global status and currently staged and modified files.
  • git log to show the latest logs. Press q to quit.
  • git show <commit1> to show the content of a commit.
  • git diff <commit1> <commit2> to show the difference between two commits.

Adding, committing and pushing

  • git add <file1> <file2> ... to add file(s) to the stage.
  • git add -p to verify all modifications and add them to the stage.
  • git commit -m "Message" -m "Description" to commit with a message. The description is optional.
  • git push to push your commits.
  • git commit --amend to fix the previous commit.
  • git push -f to push a modified history (through amend or rebasing).

If you need to fix an earlier commit than the current one, rebase.

Going between branches

  • git checkout <branch-name> to change branch
  • git checkout -b <branch-name> to create a branch and switch to it
  • git branch -D <branch-name> to delete a branch

Rebasing

  • git pull --rebase origin main to rebase to the main, or another branch.
  • git rebase -i HEAD~<number> to modify your history, reordering commits, merging them together, change their message, etc. Extremely powerful.

Vim

Summoned by default when doing amends or rebases, very powerful but very weird to use at first.
For what matters to us, only a very few commands are necessary.

  • Hit esc to go back to command mode
  • i to go edit mode
  • :m+1 or :m-2 to move a line (useful for reordering commit)
  • :q! to quit without saving
  • :wq to save and quit
